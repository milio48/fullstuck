# FullStuck.php (v0.1.0)
### The Zero-Config, Single-File, AI-Friendly PHP Framework

FullStuck.php adalah framework mikro yang dirancang untuk kecepatan pengembangan maksimal tanpa mengorbankan fitur modern. Seluruh core framework berada dalam **satu file tunggal**, membuatnya sangat mudah di-deploy ke shared hosting mana pun tanpa Composer atau folder `vendor/`.

<a name="installation"></a>
## 🚀 Quick Start & Installation

### 1. Download
Cukup unduh file `fullstuck.php` dan letakkan di root folder proyek Anda.
```text
my-project/
└── fullstuck.php
```

### 2. Jalankan Server
Gunakan PHP Built-in Server untuk development:
```bash
php -S localhost:8000 fullstuck.php
```

### 3. Setup Wizard
Buka `http://localhost:8000` di browser. Framework akan otomatis mendeteksi instalasi baru dan menampilkan **GUI Setup Wizard**.

**Fitur Unggulan Setup:**
*   **Auto-Configuration**: Membuat `fullstuck.json` dan `.htaccess` secara otomatis.
*   **AI-Friendly Mode**: Anda bisa mencentang opsi "Download documentation for AI" untuk membuat file `fullstuck_v0.1.0.md` secara lokal. Ini membantu asisten AI (ChatGPT/Claude/Copilot) memahami framework Anda secara instan.

### 4. Project Pertama Anda
Buat file `router.php` dan tambahkan rute pertama Anda:
```php
<?php
fst_get('/', function() {
    echo "<h1>Hello FullStuck!</h1>";
});
```

| Parameter | Keterangan |

<a name="routing"></a>
## 🛤️ Routing

FullStuck mendukung routing statis yang kuat dengan fitur **Strict Mode** untuk menjaga integritas arsitektur.

### 🛡️ Strict Route Detection
Framework akan menolak jika ada rute yang didefinisikan dua kali untuk metode HTTP yang sama. Hal ini mencegah bug tersembunyi di mana rute Anda "tertutup" oleh rute lain. Jika duplikat ditemukan, framework akan memberikan **Error 500** seketika.

### Contoh Routing Static
```php
fst_get('/user/{id:i}', function($id) {
    fst_json(['id' => $id]);
});

fst_group('/api/v1', function() {
    fst_get('/status', function() { fst_json(['status' => 'ok']); });
}, 'auth_middleware');
```

| Parameter | Regex | Deskripsi |
|-----------|-------|-----------|
| `{id:i}` | `([0-9]+)` | Hanya angka |
| `{name:a}` | `([a-zA-Z0-9]+)` | Alfanumerik |
| `{slug:s}` | `([a-zA-Z0-9\-]+)` | Slug (huruf, angka, strip) |

<a name="plugins"></a>
## 🔌 Plugin System

FullStuck memiliki sistem plugin berbasis file-system yang sangat sederhana namun kuat. Sistem ini dirancang agar AI dapat menambahkan fitur ke aplikasi Anda hanya dengan "menjatuhkan" satu file PHP ke dalam folder plugin.

### 1. Struktur Folder
Cukup buat folder `fst-plugins/` di root proyek Anda. Setiap file `.php` yang ada di dalam folder tersebut akan di-load secara otomatis oleh framework saat booting (setelah config di-load tapi sebelum rute dieksekusi).

### 2. Manajemen Plugin (Admin Dashboard)
Anda tidak perlu menghapus file secara manual untuk mematikan fitur. Di Admin Dashboard (`/stuck`), Anda dapat mengelola plugin secara visual:
*   **Enable/Disable**: Menonaktifkan plugin tanpa menghapus file. Framework akan menambahkan ekstensi `.disabled` agar file tersebut tidak di-load.
*   **Uninstall**: Menghapus file plugin secara permanen dari server.

### 3. AI-Friendly Workflow
Karena sistemnya berbasis file mandiri, Anda bisa memberikan instruksi ke AI seperti:
> "Buatlah plugin FullStuck yang menambahkan rute '/health' untuk mengecek koneksi database dan simpan hasilnya di fst-plugins/health_check.php."

### 4. Contoh Plugin Sederhana
`fst-plugins/hello_plugin.php`:
```php
<?php
// Plugin ini menambahkan rute global
fst_get('/plugin-test', function() {
    echo "Halo dari Plugin!";
});
```

<a name="database"></a>
## 🗄️ Database Engine

FullStuck menyediakan wrapper PDO bawaan dan Query Builder ringan yang mendukung **SQLite**, **MySQL**, dan **PostgreSQL**.

### Cross-Database Portability
Framework ini mendukung **Automatic Identifier Quoting**. Anda tidak perlu khawatir tentang perbedaan tanda petik (backtick vs double quote) antar database. Cukup tulis nama tabel/kolom secara normal, dan framework akan menyesuaikannya secara otomatis.

```php
// SELECT * FROM `users` (MySQL) atau "users" (PostgreSQL)
$users = fst_db_select('users', ['status' => 'active'], [
    'order_by' => 'created_at DESC',
    'limit' => 10
]); 
```

#### Opsi Query Builder (`$options`):
*   `select`: Nama kolom (default: `*`).
*   `order_by`: String pengurutan (misal: `id DESC`).
*   `limit`: Batas baris (Integer).
*   `offset`: Offset data (Integer).
*   `mode`: Mode pengambilan data (`ALL`, `ROW`, `SCALAR`).
```

### Konfigurasi Database
Atur di `fullstuck.json` melalui Setup Wizard atau manual:

```json
"database": {
  "driver": "pgsql",
  "pgsql": {
    "host": "localhost",
    "port": 5432,
    "dbname": "my_db",
    "username": "postgres",
    "password": "password"
  }
}
```

<a name="spa"></a>
## ✨ Zero-Config SPA

FullStuck dilengkapi fitur **Zero-Config SPA** yang mengubah aplikasi SSR (Server Side Rendering) biasa menjadi Single Page Application secara otomatis tanpa library tambahan (seperti React atau Vue).

### Cara Kerja
1.  Aktifkan `"spa": { "enabled": true }` di `fullstuck.json`.
2.  Framework akan menyuntikkan agen Javascript ringan (`fst.js`) ke setiap halaman.
3.  Setiap klik pada tag `<a>` akan dicegat. Alih-alih me-refresh halaman, agen akan mengambil konten via Fetch API.
4.  Framework hanya akan mengirimkan bagian `body` saja (HTML Clipping), sehingga navigasi terasa instan.

### Event Hook
Anda dapat menjalankan script setiap kali halaman berpindah dengan mendengarkan event `fst:load`:
```javascript
document.addEventListener('fst:load', function() {
    console.log('Halaman baru dimuat via SPA!');
    // Inisialisasi ulang library UI Anda di sini
});
```

<a name="security"></a>
## 🔐 Security & Validation

FullStuck menyertakan alat keamanan esensial untuk melindungi aplikasi Anda.

### 1. XSS Protection
Gunakan fungsi `e()` (alias dari `fst_escape`) saat menampilkan data ke HTML:
```php
<p>Halo, <?= e($user_name) ?></p>
```

### 2. CSRF Protection
Lindungi rute `POST`, `PUT`, `DELETE` dengan CSRF token:
```php
// Di View (Form)
<form method="POST">
    <?= fst_csrf_field() ?>
</form>

// Di Router
fst_post('/save', function() {
    fst_csrf_check(); // Memvalidasi token
});
```

### 3. Validasi Input
```php
$val = fst_validate(fst_request(), [
    'email' => 'required|email',
    'age'   => 'numeric|min:18'
]);

if (!$val['valid']) {
    fst_json(['errors' => $val['errors']], 400);
}
```

### 4. File Upload secara Aman
Gunakan `fst_upload` untuk menangani file upload dengan validasi tipe dan ukuran otomatis.
```php
$result = fst_upload('photo', 'uploads/profiles', [
    'max_size' => 1024, // KB
    'allowed_types' => ['jpg', 'png', 'webp']
]);

if ($result['success']) {
    echo "File diupload ke: " . $result['path'];
} else {
    echo "Error: " . $result['error'];
}
```

<a name="api-reference"></a>
## 📚 API Reference

Daftar fungsi core yang tersedia di FullStuck.php:

### Routing & HTTP
*   `fst_get|post|put|patch|delete|any($path, $callback, $middleware)`
*   `fst_group($prefix, $callback, $middleware)`
*   `fst_redirect($url, $code)`: Redirect halaman (mendukung otomatisasi SPA redirect).
*   `fst_uri()`: URI saat ini (tanpa base path).
*   `fst_method()`: Mengambil HTTP Method (`GET`, `POST`, dll).
*   `fst_status_code($code)`: Mengatur HTTP response code secara manual.
*   `fst_input($key, $default)`: Ambil data dari GET/POST/JSON.
*   `fst_request()`: Ambil seluruh data request sebagai array.
*   `fst_file($key)`: Ambil data upload file (`$_FILES`) secara aman.
*   `fst_upload($key, $folder, $options)`: Proses upload file + validasi (`max_size`, `allowed_types`).

### Response & View
*   `fst_view($path, $data)`: Render template PHP.
*   `fst_partial($path, $data)`: Alias dari `fst_view` untuk komponen kecil.
*   `fst_json($data, $status)`: Kirim response JSON & die.
*   `fst_text($string, $status)`: Kirim response teks & die.
*   `fst_abort($code, $message)`: Hentikan eksekusi dengan status error.
*   `fst_serve_static_file($path)`: Serve file asset dengan MIME type & cache header yang benar.
*   `fst_show_directory_listing($dir, $uri)`: Render UI daftar file otomatis untuk direktori.
*   `fst_extract_html_tag($html, $tag)`: Ambil isi inner HTML dari tag tertentu (berguna untuk kustomisasi SPA).

### Database (PDO Wrapper)
*   `fst_db($mode, $sql, $params)`: Query manual (Mode: `EXEC`, `ROW`, `ALL`, `SCALAR`).
*   `fst_db_select($table, $cond, $opts)`: Query Builder SELECT.
*   `fst_db_insert($table, $data)`: Query Builder INSERT.
*   `fst_db_update($table, $data, $cond)`: Query Builder UPDATE.
*   `fst_db_delete($table, $cond)`: Query Builder DELETE.
*   `fst_db_quote_ident($ident)`: Handling petik identifier otomatis (Cross-DB).

### Security & Session
*   `e($str)`: Escape HTML (XSS Protection).
*   `fst_csrf_field()`: Generate input token CSRF.
*   `fst_csrf_token()`: Ambil nilai string token CSRF aktif.
*   `fst_csrf_check()`: Validasi token CSRF.
*   `fst_validate($data, $rules)`: Validasi data input.
*   `fst_session_set|get|forget($key, $val)`: Manajemen session.
*   `fst_flash_set|get($key, $val)`: Pesan sekali baca.
*   `fst_flash_has($key)`: Cek keberadaan pesan flash.

### Config & Debugging
*   `fst_config($key, $default)`: Ambil nilai `fullstuck.json` (mendukung dot notation).
*   `fst_is_dev()`: Cek apakah environment adalah `development`.
*   `fst_dump(...$vars)`: Tampilkan dump variabel dengan UI rapi.
*   `fst_dd(...$vars)`: Dump and Die.

### SPA Advanced
*   `fst_is_spa()`: Cek apakah request berasal dari navigasi SPA.
*   `fst_spa_target()`: Ambil target container SPA (default: `body`).
